How to create your own models

Creating the model

To create a new model simply use the highlighted form. When given three options, choose between PCA, layers, and neurons to define the structure of your model and specify the desired preprocessing steps. When the model has been created you will see "Default model" change to "Model created"

If you are testing the live system, avoid using PCA due to its high computational cost for live packet capture.

It is recommended to use at least 10 neurons. Keep in mind that larger models will take longer to train and test.

Training the model

The highlighted form trains the currently in-use model.

The batch size is the number of packets given to the model at once; this lets the model find patterns in the sequence of packets and can increase accuracy.

The epochs are the number of times the data is processed through the model, but the higher the number, the greater the chance of overfitting, where the model predicts the dataset rather than identifying attacks.

Testing the model

The highlighted section shows the test button, this will test the model against a test dataset and return the accuracy. The 0% will be changed to whatever accuracy the model has when predicting the test database, be aware that this isn't reflective of the live systems performance.
